# Accessibilité

## 1. Objectif

Le logiciel doit être compréhensible même quand la lecture est difficile.

Il doit pouvoir être :
- lu
- écouté
- suivi visuellement
- utilisé dans plusieurs langues

---

## 2. Accessibilité visuelle

Le logiciel doit utiliser :
- des pictos
- des couleurs stables
- de gros boutons
- peu de texte à la fois
- des écrans simples
- des repères visuels constants

Repères de base :
- 💼 Activité
- 🏠 Vie privée
- 🧩 Aides
- 🎯 Projets
- 📊 Repères

---

## 3. Multilingue

Le projet doit au minimum prévoir :
- français
- roumain

Et plus tard, si possible :
- romanes / romani
- variantes utiles au terrain validées avec des locuteurs concernés

La traduction doit concerner :
- menus
- consignes
- explications
- messages d’aide
- audio associé

---

## 4. Audio simple

L’utilisateur doit pouvoir appuyer sur un bouton pour écouter :
- une rubrique
- une consigne
- une aide
- un message important

Le ton doit être :
- adulte
- simple
- respectueux
- non infantilisant

---

## 5. Lecture audio synchronisée

Le logiciel doit proposer une lecture audio synchronisée avec le texte.

Quand l’utilisateur appuie sur lecture :
- la phrase s’affiche
- la voix lit
- les mots se surlignent au fur et à mesure

Objectifs :
- écouter
- suivre les mots
- mieux comprendre
- repérer les mots
- apprendre progressivement

---

## 6. Trois niveaux d’accès

### Niveau 1
Picto + mot simple

### Niveau 2
Phrase courte affichée

### Niveau 3
Audio + suivi des mots

---

## 7. Phrase de synthèse

> Le logiciel doit pouvoir se lire, s’écouter, et se suivre mot à mot, afin d’aider les personnes peu lectrices à mieux comprendre sans être infantilisées.
